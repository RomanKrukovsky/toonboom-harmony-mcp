У меня уже есть существующий MCP-сервер для Toon Boom Harmony. Его не нужно создавать с нуля.

На данный момент в сервере уже есть 80+ инструментов. Твоя задача — не проектировать новый сервер, а изучить обучающий YouTube-плейлист и извлечь из него знания, паттерны, последовательности действий и сценарии, которые помогут точечно улучшать и правильнее использовать уже существующий MCP-сервер.


Главная задача:
посмотреть все видео из плейлиста как единый учебный курс по Toon Boom Harmony и понять, какую систему действий объясняет автор.

Нужно выяснить:

1. Что автор пытается научить делать.
2. Какая общая логика курса.
3. В каком порядке автор строит процесс.
4. Какие действия повторяются из видео в видео.
5. Какие действия являются ключевыми для риггинга.
6. Какие действия можно выполнять через уже существующий MCP-сервер.
7. Какие действия пока требуют ручного вмешательства.
8. Какие из существующих инструментов MCP нужно использовать для каждого этапа.
9. Какие инструменты MCP стоит доработать, но только точечно.
10. Какие новые инструкции, правила или workflow нужно добавить в документацию сервера.

Важно:
не создавай новый MCP-сервер.
не придумывай архитектуру с нуля.
не переписывай проект.
не уходи в абстрактные рассуждения.
работай только как исследователь, который изучает видео и переводит их в практические знания для уже существующего Toon Boom Harmony MCP-сервера.
