### 1. ZeBirdBrain — RIG - Rigging Basics part 1 (Harmony 17)
* **Ссылка**: [https://www.youtube.com/watch?v=rNS1c3KnTG8](https://www.youtube.com/watch?v=rNS1c3KnTG8)
* **Главная тема**: Основы построения структуры нод (Node View), иерархия Peg-to-Drawing и правила разделения трансформаций.
* **Что показано визуально**: Автор открывает окно Node View, выделяет слои рисунков (Drawing Nodes), создаёт пеги через `Cmd+P` / `Ctrl+P`, объединяет их кабелями в Composite Node. Демонстрирует окно свойства слоев (Layer Properties), переключает режим координат с 3D Path на Separate (X, Y, Z), а также показывает настройку Backdrops с выборкой цвета.
* **Цепочка действий**:
  1. Выделение Drawing нод в Node View.
  2. Нажатие `Cmd+P` для создания родитеского Peg над каждой нодой.
  3. Открытие Layer Properties пега -> Переключение в режим **Separate** (Separate Position / Scale / Rotation).
  4. Активация галочки **Can Never Enter Drawing Mode** (запрет случайного ключения слоя рисунка).
  5. Группировка связанных элементов под Backdrop (с заданием индивидуального цвета и заголовка).
  6. Соединение выходов нод в Composite ноду в режиме `Pass Through`.
* **Паттерны риггинга**: 
  - *Drawing Lock Pattern*: Никакая анимация не ставится напрямую на Drawing Node — только на Peg.
  - *Separate Mode Standard*: Пеги должны иметь раздельные каналы X/Y/Z для избежания конфликтов интерполяции кривых.
* **Ошибки и тонкости**: Автор подчеркивает ошибку начинающих — анимацию слоя Drawing. Если случайно поставить ключевой кадр на Drawing, при редактировании рисунка клейма разлетаются. Решение: блокировка Drawing слоев и анимация strictly через Peg.
* **Инсайды для MCP-сервера**: `harmony.nodes.create` при добавлении Peg должен автоматически выставлять атрибуты `POSITION.X/Y` в mode Separate и устанавливать связи `Peg -> Drawing`.
* **Добавление в поведение агента**: Добавить автоматический линтер `harmony.audit.find_missing_exposure` / `find_broken_nodes`, проверяющий отсутствие keyframes на Drawing нодах.

### 2. DrawnSean — Starting the CHARACTER RIG | Part 4
* **Ссылка**: [https://www.youtube.com/watch?v=D4s9GggdEoM](https://www.youtube.com/watch?v=D4s9GggdEoM)
* **Главная тема**: Инициализация чистого файла рига, импорт модельного листа (Model Sheet) и структурирование основных мастер-групп.
* **Что показано визуально**: Показ окон Model View и Node View. Автор перетаскивает эскиз персонажа, создаёт главную ноду `Top/Master_P`, разбивает персонажа на группы `Head_Group`, `Body_Group`, `Limbs_Group`, окрашивая ноды в контрастные цвета. Показывает работу с инспектором и выравнивание Pivot (оси вращения) с помощью инструмента `Rotate / Pivot Tool`.
* **Цепочка действий**:
  1. Импорт референсного рисунка на слой `Ref_Drawing`.
  2. Создание главного `Master_P` (Master Peg) на вершине Node View.
  3. Создание Composite нод для отдельных частей (Head, Torso, Legs, Arms).
  4. Установка оси вращения (Rotate Pivot) для каждого тела в анатомический шарнир (суставы, шея, таз).
  5. Закрепление референса на нижнем Z-слое.
* **Паттерны риггинга**: 
  - *Master Root Hierarchy*: Единственная точка контроля `Master_P` управляет всем персонажем.
  - *Anatomical Pivot Placement*: Pivot выставлен точно в центр вращения сустава на этапе базовой сборки.
* **Ошибки и тонкости**: Автор показывает, как смещенный Pivot ломает дугу движения limb при повороте. Pivot нужно ставить на виде без поворотов в нулейвой позиции (Frame 1).
* **Инсайды для MCP-сервера**: В MCP-инструмент `harmony.rig.create_character_structure` нужно заложить автоматический рассчет центроидов для суставов и простановку Pivot.
* **Добавление в поведение агента**: Агент должен требовать фиксированный Frame 1 для выставления Pivot перед добавлением кадрированной анимации.

### 3. Fabio Gioffre — The Ultimate FREE Harmony Rigging Course Pt 1
* **Ссылка**: [https://www.youtube.com/watch?v=_NgqhDkzK4g](https://www.youtube.com/watch?v=_NgqhDkzK4g)
* **Главная тема**: Анатомия профессионального пайплайна риггинга: clean-up векторов, слои Line Art / Color Art и подготовка к автоматическому патчу.
* **Что показано визуально**: Автор работает в окнах Drawing View и Node View. Переключается между вкладками **L (Line Art)**, **C (Color Art)**, **U (Underlay)**, **O (Overlay)**. Визуально показывает, как контур глаза рисуется на Line Art, а заливка на Color Art с использованием `Stroke Tool` (видимые и невидимые линии).
* **Цепочка действий**:
  1. Разделение рисунка на 4 суб-слоя (Overlay, Line Art, Color Art, Underlay).
  2. Перенос контура в Line Art, создание черновика заливки через `Create Color Art from Line Art`.
  3. Заливка цвета в Color Art.
  4. Подключение к ноде Drawing двух специальных нод: `Line Art` и `Color Art` для раздельного вывода.
* **Паттерны риггинга**:
  - *Sub-Layer Separation Pattern*: Разделение линий и заливок для создания бесшовных стыков суставов (Auto-patch).
* **Ошибки и тонкости**: Ошибка: рисовать заливку и линию на одном слое. В этом случае Auto-patch не может перекрыть линию одной деталькой под другую, создавая грязные двойные контуры.
* **Инсайды для MCP-сервера**: Команда `harmony.drawings.create_layer` должна поддерживать авто-генерацию шаблонов суб-слоев (Line Art / Color Art separation).
* **Добавление в поведение агента**: Правило: всегда проверять наличие независимых суб-слоев при создании элементов суставов (локти, колени, плечи).

### 4. Onion Skin — Rigging with Bones in Toon Boom Essentials
* **Ссылка**: [https://www.youtube.com/watch?v=XJdsJKxR4J4](https://www.youtube.com/watch?v=XJdsJKxR4J4)
* **Главная тема**: Использование костных деформеров (Bone Deformers) для быстрой сборки базового рига.
* **Что показано визуально**: Инструмент `Rigging Tool` (кликом по суставам образуется цепочка костей). Отображение зеленых и красных маркеров суставов. Изменение зоны влияния костей (Radius) в Viewport.
* **Цепочка действий**:
  1. Выбор ноды элемента (например, хвост или рука).
  2. Активация `Rigging Tool` в режиме **Bone**.
  3. Клик в основании детали -> клик в сочленении -> клик на кончике.
  4. Регулировка кругов влияния (Zone of Influence) для плавного изгиба меша.
  5. Добавление `Deformation Composite` ноды.
* **Паттерны риггинга**:
  - *Linear Bone Chain*: Кости подходят для простых органических жестких или полужёстких деталей (хвосты, уши, простые конечности).
* **Ошибки и тонкости**: Слишком большой радиус зоны влияния кости вызывает паразитное искажение соседних рисунков. Автор показывает уменьшение радиуса зоны влияния в свойствах деформера.
* **Инсайды для MCP-сервера**: Реализовать в `harmony.rig.create_deformers` тип `Bone` с параметрами точек и радиусов.
* **Добавление в поведение агента**: Использовать `Bone` деформеры только для вторичных элементов, отдавая предпочтение `Curve` или `Envelope` для основных рук и ног.

### 5. DrawnSean — RIGGING Sonnie's ARM | Part 9
* **Ссылка**: [https://www.youtube.com/watch?v=mQ_AbSm2Eyc](https://www.youtube.com/watch?v=mQ_AbSm2Eyc)
* **Главная тема**: Комплексный риг руки с авто-патчем (Auto-patch), плечом, предплечьем и кистью с заменой рисунков (Substitutions).
* **Что показано визуально**: Окно Node View с разветвленной сетью нод: `Bicep_D`, `Forearm_D`, `Hand_D`, ноды `Auto-Patch`, `Cutter`, `Peg`. В окне Timeline показывается панель **Library / Drawing Substitutions** при переключении кадров кисти.
* **Цепочка действий**:
  1. Иерархическое соединение: `Shoulder_P -> Bicep_P -> Forearm_P -> Hand_P`.
  2. Подключение ноды `Auto-patch` от `Forearm_D` к верхнему порту `Bicep_D` через Composite.
  3. Настройка выравнивания кругового сустава (Round Joint Alignment).
  4. Импорт набора рисунков кисти (Hand drawings: FIST, OPEN, POINT, GRAB).
  5. Регистрация заменителей рисунков (Drawing Substitutions).
* **Паттерны риггинга**:
  - *Auto-patch Joint Pattern*: Заливка предплечья накладывается поверх контура бицепса, создавая идеальный сустав без швов при вращении.
  - *Hand Swap Library Pattern*: Пег кисти управляет положением, а Drawing Node содержит набор кадров-субституций.
* **Ошибки и тонкости**: Заливка Color Art на бицепсе и предплечье должна именоваться одинаковой палитровым цветом. Если цвета палитры различаются, Auto-patch дает сбой цвета на стыке.
* **Инсайды для MCP-сервера**: `harmony.rig.create_cutout_hierarchy` должен создавать связку `Auto-patch` нод автоматизировано для сочленений плечо-локоть-кисть и бедро-колено-стопа.
* **Добавление в поведение агента**: Всегда генерировать стандартизированную библиотеку кистей рук с предсказуемыми именами файлов субституций (Hand_Open, Hand_Fist, etc.).

### 6. M Positano — Toonboom Harmony - Articulation
* **Ссылка**: [https://www.youtube.com/watch?v=da8HLRdP0eM](https://www.youtube.com/watch?v=da8HLRdP0eM)
* **Главная тема**: Анализ правильной анатомической артикуляции и наложения слоев Z-depth на стыках персонажа.
* **Что показано визуально**: Автор показывает манипуляции в окна Camera View при включенном режиме `Show Layer Z-Ordering`. Двигает детали вперед/назад с помощью `Alt+Down / Alt+Up` (сдвиг по оси Z на микроскопические значения 0.001B).
* **Цепочка действий**:
  1. Рисование перекрывающих полукругов на концах конечностей.
  2. Выставление точной совпадающей точки вращения (Pivot) двух перекрывающихся окружностей.
  3. Тонкая настройка параметра Offset Z в свойствах Peg ноды (`0.0001` для вывода деталей на передний план без изменения порядка нод в Composite).
* **Паттерны риггинга**:
  - *Perfect Circle Joint Rule*: Концы конечностей на стыках сустава рисуются идеальными окружностями с одинаковым радиусом.
  - *Micro Z-Offset Pattern*: Использование минимального Z-смещения для контроля перекрытия слоев при сохранении чистой структуры Composite.
* **Ошибки и тонкости**: Автор предостерегает от чрезмерного использования сдвигов по Z. Если сдвиги по Z слишком велики, при повороте персонажа на 180 градусов слои выворачиваются наизнанку.
* **Инсайды для MCP-сервера**: Позволить MCP-инструменту `harmony.nodes.set_attr` точно управлять `position.z` значением в диапазоне `[-0.01, 0.01]`.
* **Добавление в поведение агента**: Соблюдать правило Micro Z-Offset — использовать Z-сдвиг только в пределах тысячных долей.

* **Добавление в поведение агента**: Запустить эвристическую проверку топологии схемы перед переходом к фазе создания деформеров.

### 9. DrawnSean — AUTOPATCHES and CUTTERS | Part 8
* **Ссылка**: [https://www.youtube.com/watch?v=dsZnx8ZDBkk](https://www.youtube.com/watch?v=dsZnx8ZDBkk)
* **Главная тема**: Маскирование и авто-патчинг: продвинутая магия масок для бровей, глаз и рукавов.
* **Что показано визуально**: Работа с нодой `Cutter` и инвертированным маскированием (`Invert Cutter`). Показывается подсоединение маскирующего рисунка к левому порту `Cutter`, а маскируемого элемента — к правому порту.
* **Цепочка действий**:
  1. Добавление ноды `Cutter` из Node Library.
  2. Подключение элемента (например, зрачок) к правому порту `Cutter`.
  3. Подключение формы глазного яблока (`Eye_BG`) к левому порту (Matte Port).
  4. Двойной клик на `Cutter` для включения режима **Inverted** (если зрачок должен оставаться строго внутри глаза).
  5. Для рукава: создание Auto-Patch от предплечья и резание его по маске плеча.
* **Паттерны риггинга**:
  - *Eye Clipping Pattern*: `Pupil -> Cutter (Matte: Eye_White) -> Composite`.
  - *Sleeve Masking Pattern*: Авто-патч заправляет руку внутрь рукава без вылезания контура за границы.
* **Ошибки и тонкости**: Частая ошибка — перепутать левый и правый порт ноды `Cutter`. Левый порт — это ВСЕГДА маска (Matte), правый — изображение (Image).
* **Инсайды для MCP-сервера**: Tool `harmony.nodes.connect` должен явно подписывать имена портов: `matte_input` и `image_input` для ноды `Cutter`.
* **Добавление в поведение агента**: Жесткое правило линтера: при создании связи с `Cutter` сверять роли нод (Маска -> Левый порт, Рисунок -> Правый порт).

## ЧАСТЬ 2. Общий документ и глобальный синтез знания (Global Rigging Synthesis)

### 1. Общая цепочка создания рига от начала до конца (Full End-to-End Pipeline)

На основе визуального анализа 28 источников сформирован эталонный 10-этапный пайплайн разработки персонажного рига в Toon Boom Harmony:

```
[1. Asset Prep & Cleanup] -> [2. Node View Topo Init] -> [3. Layer Sub-Separation] -> [4. Peg Hierarchy & Pivots]
                                                                                               |
[8. Master Controllers]  <- [7. Deformer Integration] <- [6. Auto-Patch & Cutters]  <- [5. Articulation & Z-Ordering]
         |
         v
[9. Stress Testing & Rest Pose] -> [10. Template Export (.tpl)]
```

1. **Импорт и подготовка эскизов (Asset Prep)**: Импорт модельного листа, выравнивание по сетке, масштабирование под стандарт сцены (Frame Height).
2. **Инициализация топологии Node View (Node View Topo Init)**: Создание корневого `Master_P`, главных композитов (`Composite_PassThrough`) и цветовых подложек (Backdrops) для Head, Torso, Arms, Legs.
3. **Разделение суб-слоев (Layer Sub-Separation)**: Отрисовка элементов с явным разделением на **Line Art** (контуры) и **Color Art** (заливки).
4. **Построение иерархии Peg и выставление осей (Peg Hierarchy & Pivots)**:
   - Создание индивидуального родительского Peg над каждым Drawing слоем.
   - Переключение атрибутов пегов в **Separate Mode** (Separate X/Y/Z).
   - Выставление точек вращения (Pivots) строго в анатомические центры суставов на Frame 1.
5. **Артикуляция и Z-ранжирование (Z-Ordering & Articulation)**: Настройка бесшовных круговых суставов (Perfect Circle Joints) и тонкая регулировка `Micro Z-Offset` (`0.0001B` .. `0.001B`).
6. **Маскирование и бесшовные стыки (Auto-Patch & Cutters)**:
   - Подключение `Auto-patch` нод для суставов конечностей (плечи, локти, колени).
   - Сборка модулей глаз и рта с нодами `Cutter` (Зрачок -> Right Port, Вектор глаза -> Left Port / Matte).
7. **Интеграция систем деформации (Deformers)**:
   - Наложение `Curve Deformers` на гибкие конечности и волосы.
   - Наложение `Envelope Deformers` на торс, одежду и контур головы.
   - Подключение `Kinematic Output` нод для всех прикрепленных аксессуаров (браслеты, пуговицы, часы).
8. **Сборка Master Controllers (Master Controllers Setup)**:
   - Раскадровка 9 ключевых фаз поворота головы (Grid 3x3) и 5-8 фаз поворота тела (360 Turnaround).
   - Генерация виджетов через `Grid Wizard` и `Slider Wizard`.
   - Расстановка GUI-виджетов по краям экрана.
9. **Стресс-тестирование и очистка (Stress Testing & Rest Pose Reset)**:
   - Проверка изгибов на 180°, проверка отсутствия дыр в веторах.
   - Очистка таймлайна от анимационных ключей, сброс всех пегов в **Rest Pose** (`R`).
10. **Экспорт шаблона (Template Export)**: Группировка рига в единую мастер-ноду и сохранение в библиотеку шаблонов (`.tpl`).

### 3. Список профессиональных советов (Pro Rigging Secrets)

1. **Правило Separate Mode**: Всегда переключать координаты Peg нод в режим `Separate` (X, Y, Z). Режим `3D Path` создает непредсказуемые паразитные дуги при анимации.
2. **Принцип единого ракурса на Frame 1**: Все оси вращения (Pivots) и базовые формы должны выставляться strictly на капре 1. Запрещено менять Pivot на кадрах с анимацией.
3. **Идеальная окружность сустава**: Суставы локтей и коленей на слое Line Art рисуются как соприкасающиеся идеальные круги. При вращении авто-патч перекрывает круговую заливку без возникновения изломов.
4. **Micro Z-Offset вместо перекоммутации кабелей**: Используйте микро-сдвиги по оси Z (`Alt+Down` / `0.0001B`) для локального вывода слоев вперед/назад, не усложняя Composite ноды лишними кабелями.
5. **Префиксная стандартизация наименований нод**:
   - `_P` — Peg (например, `Head_P`)
   - `_D` — Drawing Layer (`Head_D`)
   - `_Comp` — Composite Node (`Head_Comp`)
   - `_Def` — Deformer Group (`Head_Def`)
   - `_MC` — Master Controller Node (`Head_Turn_MC`)
6. **Цветовое зонирование Backdrops**:
   - **Зеленый**: Голова и лицо
   - **Синий**: Торс и корпус
   - **Желтый/Оранжевый**: Руки
   - **Фиолетовый**: Ноги
   - **Красный**: Мастер-контроллеры и управляющие пеги

### 4. Типичные ошибки и способы их исправления (Common Pitfalls & Fixes)

| Типичная ошибка | Последствия | Способ исправления |
| :--- | :--- | :--- |
| **Анимация на Drawing ноде** | Ключевые кадры попадают на слой рисунка, разлетаются при смене фазы. | Включить атрибут `Can Never Enter Drawing Mode` на всех Drawing нодах. |
| **Перепутаны порты Cutter** | Изображение режется наоборот или полностью исчезает с экрана. | Маска (Matte) -> Левый порт, Обрезаемый объект (Image) -> Правый порт. |
| **Отсутствие Kinematic Output** | Браслеты/часы улетают от руки при изгибе деформера. | Вставить ноду `Kinematic Output` внутри `Deformation Group` и привязать к ней пег аксессуара. |
| **Разные цвета палитры на Auto-patch** | Появление темных линеек и швов на сочленении суставов. | Назначать строго один цвет из Palettes для Color Art обеих деталей сустава. |
| **Хаотичные Pivot на разных кадрах** | Персонаж "прыгает" и деформируется при повороте джойстика Master Controller. | Выровнять координаты Pivot по единой сетке на всех ракурсах Turnaround на капре 1. |

### 5. Карта визуальных действий для автоматизации (Visual-to-MCP Mapping)

Маппинг экранных манипуляций риггера в вызовы API MCP-сервера `toonboom-harmony-mcp`:

| Визуальное действие риггера в UI | Действие в Harmony GUI / QScript | Вызов инструмента `toonboom-harmony-mcp` |
| :--- | :--- | :--- |
| Нажатие `Cmd+P` над выделенной нодой | `Selection.addPeg()` | `harmony.nodes.create` (type: `PEG`, target: `node_path`) |
| Перетаскивание кабеля от ноды A к B | `node.link(nodeA, 0, nodeB, 0)` | `harmony.nodes.connect` (`from_node`, `to_node`, `port`) |
| Клик инструментом Pivot Tool по Viewport | `node.setTextAttr(peg, "PIVOT.X", frame, valX)` | `harmony.nodes.set_attr` (`node`, `attr: PIVOT.X/Y`, `value`) |

### 7. Вывод: Как эти знания улучшат MCP-сервер Toon Boom Harmony

Анализ 28 практических обучающих материалов позволил выявить фундаментальные векторы развития сервера `toonboom-harmony-mcp`:

1. **Расширение возможностей существующего инструментария**:
   - В инструмент `harmony.nodes.create` необходимо добавить опции авто-конфигурации каналов `Separate X/Y/Z` и установки защиты `Can Never Enter Drawing Mode`.
   - В `harmony.nodes.connect` внедрить именованную адресацию портов (`matte_port`, `image_port` для `Cutter`, `pass_through` для `Composite`).

2. **Создание новых высокоуровневых инструментов автоматизации**:
   - `harmony.rig.create_autopatch_joint`: Специализированный инструмент, принимающий 2 конечности и автоматически выстраивающий всю нодовую обвязку из `Auto-patch`, `Line Art`, `Color Art` и `Composite`.
   - `harmony.rig.attach_kinematic_accessory`: Инструмент автоматического внедрения `Kinematic Output` ноды между деформируемым родителем и аксессуаром.

3. **Интеллектуальные правила и линтеры для агента**:
   - Агент получает строгое древо верификации: запрет на создание незафиксированных осей Pivot, запрет на неструктурированные имена нод, автоматическую укладку нод в Node View после каждого действия.
   - Автоматический прогон тестов артикуляции (Stress Testing) гарантирует, что генерируемые агентом риги будут на 100% готовы к коммерческому производству анимации.
