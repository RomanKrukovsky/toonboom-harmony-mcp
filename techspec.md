Создание сервера Toon Boom Harmony MCP с полной производственной функциональностью
Введение

Toon Boom Harmony Server — это сетевой слой для Harmony, который хранит сцены и активы в централизованной базе данных и позволяет множеству художников работать над одним проектом в реальном времени. Сервер предоставляет инфраструктуру для создания рабочих сред (environments), проектов (jobs) и сцен, предлагает систему контроля версий и очереди для векторизации и рендеринга, а также обеспечивает совместный доступ к палитрам и библиотекам. Удалённые художники могут получать доступ через веб‑интерфейс WebCC, скачивать сцены, работать локально и загружать изменения обратно, не будучи подключёнными к локальной сети студии.

В новом сленге сообщества AI термин «MCP‑сервер» часто используется для обозначения сервера, который принимает запросы от интеллектуальных агентов (например, GPT) и предоставляет контекст о приложении (Model Context Protocol). В случае Harmony под MCP подразумевается режим скриптового сервера Control Center, когда Control Center запускается с флагом -script, принимает инструкции по Telnet и позволяет программно управлять базой данных и производственным процессом. Такой подход открывает возможности для автоматизации и интеграции с внешними системами, включая AI‑агентов.

Ниже приведено пошаговое руководство по развёртыванию и настройке Harmony Server с максимальной функциональностью, а также советы по использованию скриптовых интерфейсов (Qt Script и Python) и вспомогательных компонентов, таких как WebCC, Batch Processing и Producer.

1. Основы Harmony Server
Централизация сцен и активов. Harmony Server хранит сцены и рисунки в базе данных и позволяет клиентским компьютерам открывать их по сети. Сервер позволяет создавать, управлять, векторизовать и рендерить сцены, загружать и сохранять их прямо на сервере, совместно использовать палитры, рисунки и шаблоны, а также отслеживать стадии утверждения.
Клиентские приложения. Пакет Harmony Server включает приложения Harmony, Play, Paint, Scan, Control Center и WebCC. Рабочие сцены создаются через Control Center (при использовании сервера), а не напрямую в приложении Harmony.
Рабочие среды и структуры. Control Center использует иерархию environment → job → scene. Environment определяет корневой путь и файл‑систему. Job содержит набор сцен, а каждая сцена может иметь несколько версий.
2. Установка и конфигурация базы данных
2.1 Создание базы данных
Windows. Используйте Configuration Wizard. В списке задач выберите «Database», затем «Server/Standalone» и укажите диск для хранения базы; мастер создаст каталог USA_DB на выбранном диске. Далее укажите имя пользователя (по умолчанию usabatch) и пароль, и примите изменения, чтобы запустить службу базы данных. После завершения вы сможете войти в Control Center под пользователем usabatch для создания сцен и пользователей.
macOS и Linux. Процедура аналогична, но вместо мастера используются скрипты установки. Настроенная база должна быть размещена в каталоге USA_DB.
Общие требования. Для функционирования сервера необходимо:
Создать структуру базы данных и каталог данных сцен.
Настроить общий доступ к этим каталогам: Samba для Windows, NFS для macOS/Linux.
Запустить службу базы данных при старте системы.
2.2 Настройка общего доступа и системы
Общий доступ к файлам. Клиентские машины должны видеть путь к USA_DB через сетевой путь (NFS или Samba). Windows‑клиенты могут работать с Linux‑сервером через Samba и специальный link server, который создаёт символические ссылки для Windows. Разрешите доступ в файрволе для сервера, link‑сервера и файловых служб.
Порты. По умолчанию база данных слушает на TCP‑порт 5678 (управление) и 5680 (общение с клиентами). Для Linux откройте эти порты командой iptables или firewall-cmd. На Windows откройте порт 5680 через «Защитник Windows»; номер порта можно изменить, отредактировав файл Dbserver.conf в каталоге USA_DB.
Безопасность. Разрешите входящие соединения только с доверенных сетей; для выхода за пределы студии лучше устанавливать WebCC на отдельном веб‑сервере, а не напрямую публиковать базу данных в Интернет.
2.3 Запуск Control Center
Запустите Control Center на сервере или любом клиенте. Перед входом появится диалог «Database Login»; введите имя пользователя usabatch. Создайте своих пользователей, но не удаляйте usabatch, т.к. он используется для пакетных задач.
Через интерфейс вы можете создавать environments, jobs и scenes, импортировать и экспортировать пакеты сцен, контролировать очереди векторизации и рендеринга и управлять правами пользователей.
3. Управление производственным процессом через Control Center

Control Center является ядром администрирования базы данных. Он позволяет:

Создавать и изменять пользователей. Класс ControlCentre предоставляет методы addUser, deleteUser, modifyUser и verifyUser.
Работа с environments. Методы addEnvironment, deleteEnvironment, modifyEnvironment и verifyEnvironment позволяют управлять средами. Каждая среда определяет путь в базе, файловую систему (NFS/Samba) и параметры блокировок.
Создавать jobs и scenes. Через методы addJob, deleteJob, modifyJob и jobs можно управлять проектами, а addScene, deleteScene, scenes, lockedScenes и lockedSceneVersions позволяют управлять сценами. Вы также можете получать список заблокированных сцен, чтобы видеть, кто в них работает.
Импорт и экспорт. В Control Center есть функции importScene и exportScene (см. пример ниже) для переноса сцен между базами или резервного копирования.
Очереди векторизации и рендеринга. Вы можете просматривать статус очередей для конкретного job. Batch Processing (см. ниже) обеспечивает удалённую векторизацию и рендеринг, разгружая рабочие станции.
4. Скриптовый интерфейс Control Center (MCP‑режим)
4.1 Общие сведения

Harmony предлагает несколько скриптовых интерфейсов:

Harmony Scripting (Qt Script). Основной API для автоматизации в рамках приложения, создания диалогов и выполнения пакетных операций.
Extended Scripting Interface. Предоставляет дополнительные модули для редактирования рисунков, создания инструментов и запуска Python‑скриптов.
Script Node. Узел, который принимает входные изображения или пеги, обрабатывает их пользовательским скриптом и выдаёт результат, позволяя создать собственные процедурные узлы в графе.
Control Center Scripting. Интерфейс для управления базой данных; скрипты запускаются либо в пакетном режиме, либо через сервер Telnet.
4.2 Пакетный режим (-runScript)

Скрипт (файл .js) выполняется командой:

Controlcenter -runScript <путь_к_скрипту> -user <имя_пользователя>

Скрипт должен определить функцию, а затем вызвать её в конце; скрипты для пакетного режима не должны содержать токены TB_BeginScript и TB_EndScript.
4.3 Telnet‑сервер (-script, MCP‑режим)

Запуск Control Center как Telnet‑скрипт‑сервер делает его аналогом MCP‑сервера. Сервер слушает на порт, указанный в переменной TOONBOOM_REMOTE_SCRIPT или через аргумент -tcpPort. Пример:

# Linux/macOS
export TOONBOOM_REMOTE_SCRIPT=1234
controlcenter -script

# Windows
SET TOONBOOM_REMOTE_SCRIPT=1234
controlcenter -script

После запуска вы можете подключаться по Telnet (telnet server_ip 1234) и отправлять скрипты. Скрипт должен быть заключён между TB_BeginScript и TB_EndScript. В теле скрипта доступны те же классы, что и в пакетном режиме. Пример, который печатает все версии сцен в указанном environment/job, выглядит так:

TB_BeginScript
list = ControlCentre.environments();
for (var i = 0; i < list.length; i++) {
    var env = list[i];
    if (env.name == "Test") {
        var jobList = ControlCentre.jobs(env);
        for (var j = 0; j < jobList.length; j++) {
            var job = jobList[j];
            if (job.name == "a") {
                var sceneList = ControlCentre.scenes(job);
                for (var k = 0; k < sceneList.length; k++) {
                    var scene = sceneList[k];
                    var versions = ControlCentre.versions(scene);
                    for (var l = 0; l < versions.length; l++) {
                        var ver = versions[l];
                        ControlCentre.printToConsole(ver.toString(true));
                    }
                }
            }
        }
    }
}
var log = ControlCentre.messageLog();
ControlCentre.printToConsole(log);
TB_EndScript

Полезно завершать скрипты чтением messageLog и выводом сообщений, чтобы видеть ошибки и результат.

Преимущества MCP‑режима:

Скрипты выполняются непосредственно на сервере, что повышает производительность при доступе к базе данных.
Можно интегрировать Telnet‑сервер с внешними агентами или написать шлюз (например, REST‑API), который будет принимать команды и передавать их Control Center через Telnet.
Сервер позволяет практически полностью автоматизировать создание environments, jobs, импорт сцен, обновление статусов, управление пользователями и т.п.
5. Python‑интерфейс Harmony

Начиная с версии 24 Harmony имеет полноценный Python API. Python‑модуль (ToonBoom.harmony) представляет текущий сеанс и проект в виде объектной модели; его можно использовать как внутри приложения, так и во внешнем интерпретаторе.

5.1 Внутренний Python Console
В самой Harmony доступны встроенные окно Python Script Console (Windows → Python Script Console). Эта консоль предоставляет постоянную сессию Python 3.9 и доступ к модулю ToonBoom.harmony.
Пример использования:
from ToonBoom import harmony
current_session = harmony.session()      # активная сессия
project = current_session.project       # текущий проект
print("Current Project:", project.project_path)
Пакет включает библиотеку PySide6, позволяющую создавать собственные окна и панельки в интерфейсе Harmony, и вы можете комбинировать стандартные Python‑библиотеки (requests, numpy и т.д.) с Harmony API.
5.2 Внешний интерпретатор
Внешние скрипты позволяют выполнять операции без графического интерфейса — полезно для пакетной обработки и интеграции. Добавьте путь к python-packages (например, $HARMONY_INSTALL/win64/bin/python-packages) в sys.path, импортируйте модуль ToonBoom.harmony, откройте оффлайн проект и получите объект сессии.
Из внешнего скрипта вы можете: открывать/закрывать проекты, изменять сцены, управлять узлами, экспортировать изображения, сохранять проект и т. д.
Следует учитывать, что объекты сцены должны модифицироваться в главном потоке Harmony. В Python API есть методы run_on_main() и thread_lock() для синхронизации.
6. WebCC — удалённая работа

WebCC — это веб‑сервер, который размещает базу данных Harmony в Интернете и позволяет пользователям просматривать каталог, скачивать сцены и заливать изменения без прямого подключения к локальной сети. Пользователям нужен только браузер и установленная копия Harmony Stand Alone для редактирования сцен. WebCC рекомендуется разворачивать на отдельном веб‑сервере и давать доступ либо только внутри студии, либо через безопасный шлюз.

7. Batch Processing — векторизация и рендеринг

Harmony Server позволяет настроить рендер‑ферму, чтобы разгружать рабочие станции:

Векторизация. Преобразует отсканированные растровые рисунки в векторный формат. В рамках традиционного производства рисунки попадают в очередь векторизации автоматически, но их можно отправлять вручную через Control Center, например, если изображения были импортированы сторонними способами.
Рендеринг (композитинг). Создаёт финальные кадры и видеофайлы. Процесс ресурсозатратный, поэтому его лучше выполнять на выделённых машинах. Художник отправляет сцену в очередь рендеринга, а настроенные машины берут задачи, когда они свободны.
Настройка. Для каждой машины добавьте её имя в список машин в базе данных, установите службу batch‑processing, затем создайте расписание — например, выполнять задачи ночью, чтобы не мешать работе художников. Обязательно добавьте путь к утилитам Harmony в переменную PATH, чтобы запускать команды без длинных путей.
8. Producer — управление производством

Для полного контроля над производством Toon Boom предлагает Producer 22. Этот инструмент интегрируется с Harmony и Storyboard Pro и обеспечивает:

Отслеживание задач, активов и сцен. Producer отображает статус кадров, позволяет назначать задания, контролировать сроки и распределять ресурсы.
Отчётность и аналитика. Панели управления, отчёты по задачам, прогрессу и производительности команд; кастомизируемые триггеры и автоматические уведомления.
Многопользовательская среда. Поддерживает несколько проектов и команд, версии сцен, метки, предварительные просмотры и управление пользователями.
Гибкое развёртывание. Producer можно разместить локально или в облаке, что упрощает управление удалёнными командами.

Producer не является частью Harmony Server, но вместе с ним обеспечивает полный цикл: планирование, производство, отслеживание и аналитику.

9. Практические рекомендации
Разделите роли. Выделите администратора сервера, который отвечает за установку, бэкапы и безопасность. Художники и супервайзеры должны использовать Control Center только для своих задач.
Используйте версии сцен. Никогда не редактируйте файлы в обход сервера; версии позволяют откатываться и отслеживать изменения.
Бэкапы и зеркалирование. Делайте регулярные бэкапы базы (USA_DB) и сцены, настройте зеркальный сервер для отказоустойчивости.
Интеграция с AI (MCP). Если вы хотите, чтобы AI‑агенты управляли Harmony, создайте обёртку вокруг Telnet‑серверa: скрипт на Node.js или Python, который принимает HTTP‑запросы, генерирует скрипты для Control Center и отправляет их через Telnet. Помните, что для безопасности следует ограничивать набор допустимых команд.
Расширение через Python и Qt Script. Автоматизируйте повторяющиеся задачи: генерация кадров, настройка нодов, импорт экспозиционных листов. Создавайте GUI‑панели с PySide для упрощения работы художников.
Планирование ресурсов. Настройте расписание batch‑processing таким образом, чтобы рендер‑ферма не мешала рабочему процессу; в Producer назначайте приоритеты и контролируйте загрузку.
Заключение

Создание «крутого» MCP‑сервера для Toon Boom Harmony подразумевает развёртывание полноценного Harmony Server (база данных, Control Center, WebCC и Batch Processing), а затем включение режима скриптового сервера в Control Center. Этот режим позволяет внешним программам отправлять команды для автоматизации производства и интеграции с AI‑агентами. Дополнительные скриптовые интерфейсы (Qt Script и Python) дают возможность создавать собственные инструменты, узлы и графический интерфейс. В связке с инструментом Producer вы получаете полную систему управления проектом: от хранения рисунков до планирования задач и анализа производительности. Такой комплекс обеспечивает гибкость, масштабируемость и готовность к будущим интеграциям, включая автоматизацию через AI.
