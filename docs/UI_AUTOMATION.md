# UI Automation Layer & Visual State

Для выполнения действий, которые отсутствуют в официальном JavaScript/Python API Toon Boom Harmony, автопилот использует слой **UI Automation**.

## Принцип работы

Автопилот не является простым автокликером. Вместо этого он действует по циклу **восприятие -> планирование -> действие -> проверка**:

1. **Захват состояния**: Инструмент делает скриншот активного монитора с помощью `ScreenshotAdapter`.
2. **Семантический анализ**: `VisualStateEngine` парсит изображение для поиска панелей (Camera View, Node View, Timeline) и открытых диалоговых окон.
3. **Определение координат**: Метод `locateElement` возвращает относительные координаты для кликов. Координаты вычисляются на основе шаблонов кнопок и панелей, что делает систему устойчивой к изменению разрешений экрана.
4. **Выполнение клика/хоткея**: Нажатие клавиш или движение мыши передается через `UIAutomationAdapter`.
5. **Проверка**: После каждого действия вызывается проверка `verifyState`. Если состояние не подтверждено, запускается цепочка восстановления.

## Поддержка операционных систем

* **macOS**: Использует `screencapture` для создания снимков экрана.
* **Windows**: Задействует PowerShell скрипт для быстрого захвата первичного монитора.
* **Linux**: Требует `scrot` или `import`.

Для симуляции процесса при локальном тестировании без запущенной Harmony используется встроенный **Mock UI Driver**, который эмулирует изменение заголовков окон и слоев таймлайна.
