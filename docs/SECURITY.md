# Правила безопасности

Для защиты от несанкционированного удаления баз данных, стирания сцен или атак на файловую систему хоста через некорректно сформулированные промпты AI-агентов, MCP-сервер Harmony внедряет строгие правила безопасности.

## 1. Песочница путей (`HARMONY_ALLOWED_ROOTS`)

Любая операция, запрашивающая пути к файлам (например, открытие проекта `.xstage`, импорт растровых ресурсов, указание путей вывода рендеринга), проходит проверку:
- Путь приводится к абсолютному каноническому виду, исключая трюки обхода директорий (`../`).
- Итоговый путь должен начинаться с одной из разрешенных корневых директорий, перечисленных в `HARMONY_ALLOWED_ROOTS`.
- Попытка доступа к запрещенным папкам блокируется с ошибкой `PATH_NOT_ALLOWED`.

## 2. Защита от инъекций команд

- Сервер **никогда** не выполняет сырые строки команд в оболочке (shell).
- Все запуски внешних процессов (например, утилиты `Controlcenter`, `HarmonyPremium` и `python`) осуществляются с помощью методов Node.js `execFile` или `spawn` с передачей параметров в виде массива аргументов. Это исключает разделение аргументов и инъекцию шелл-кода.

## 3. Деструктивные действия

Опасные действия, классифицируемые как деструктивные (удаление узлов графа, переименование сцен в базе, создание администраторов):
- По умолчанию выключены, если переменная `HARMONY_ALLOW_DESTRUCTIVE` не равна `true`.
- Требуют передачи параметров `confirm: true` и точной строки подтверждения `"Я понимаю, что это действие изменит базу данных Harmony"`.

## 4. Режим симуляции (Dry-Run)

- Все инструменты создания и изменения сущностей поддерживают параметр симуляции выполнения `dryRun`.
- Состояние по умолчанию контролируется переменной `HARMONY_DRY_RUN_DEFAULT` (рекомендуется выставлять в `true`).

## 5. Аудит операций

- Все поступающие запросы от агента, их параметры и результаты выполнения пишутся в структурированном JSONL-виде в файл логов в папке `HARMONY_LOG_DIR`.
- Параметры паролей и секретных токенов автоматически маскируются перед записью в лог-файл.
