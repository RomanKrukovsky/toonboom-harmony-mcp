# Решение проблем (Troubleshooting)

Руководство по диагностике и решению типовых ошибок сервера автоматизации Harmony.

## 1. Ошибка соединения с Control Center (`CONTROL_CENTER_UNREACHABLE`)

- **Причина**: Сервер удаленных сценариев Control Center не запущен на хосте базы данных или доступ к нему заблокирован.
- **Решение**:
  1. Убедитесь, что сервер запущен с помощью команды, полученной из `harmony.cc.start_script_server_command`.
  2. Проверьте правильность значений переменных `HARMONY_CC_HOST` и `HARMONY_CC_PORT` в вашем файле `.env`.
  3. Проверьте настройки брандмауэра и убедитесь, что порт `1234` открыт для входящих TCP-соединений.

## 2. Ошибка импорта библиотек Python (`PYTHON_API_UNAVAILABLE`)

- **Причина**: Модуль `ToonBoom.harmony` отсутствует в путях импорта Python.
- **Решение**:
  1. Проверьте переменную `HARMONY_PYTHON_PACKAGES` в файле `.env`. Она должна указывать на папку внутри установки Harmony, содержащую пакет `ToonBoom/harmony`.
  2. Запустите `harmony.detect_installation` для просмотра автоматически определенных сервером путей.
  3. Если вы используете виртуальное окружение Python, укажите путь к интерпретатору в переменной `PYTHON_BIN` (например, `PYTHON_BIN=/usr/local/bin/python3`).

## 3. Ошибка безопасности пути (`PATH_NOT_ALLOWED`)

- **Причина**: Запрошенный файл или папка назначения находятся вне списка разрешенных корневых директорий.
- **Решение**:
  1. Добавьте необходимые каталоги в список через запятую в переменной `HARMONY_ALLOWED_ROOTS` вашего файла `.env`.
  2. Убедитесь, что передаете абсолютные пути.

## 4. Истечение таймаута выполнения (`SCRIPT_TIMEOUT`)

- **Причина**: Выполнение скрипта превысило заданное значение `HARMONY_SCRIPT_TIMEOUT_MS`.
- **Решение**:
  1. Увеличьте значение времени таймаута в файле `.env` (например, `HARMONY_SCRIPT_TIMEOUT_MS=30000` для 30 секунд).
  2. Проверьте стабильность локального сетевого соединения с сервером базы данных.
