# Commercial Demo Execution Guide

В проект встроен демонстрационный сценарий (Commercial Demo), призванный продемонстрировать возможности Harmony Autopilot клиентам или руководству без необходимости ручной настройки.

## Шаг 1. Настройка окружения
Для запуска демонстрационного сценария в режиме симуляции убедитесь, что в переменных окружения включен симулятор:
```bash
export HARMONY_UI_SIMULATE=true
```

## Шаг 2. Запуск автопилота
Вызовите MCP-инструмент сборки сцены, передав путь к демо-плану:
* **Tool**: `harmony.autopilot.run_scene_plan`
* **Arguments**:
  ```json
  {
    "scenePlanPath": "examples/commercial-demo/scene_plan.json",
    "dryRun": false
  }
  ```

## Шаг 3. Чтение логов выполнения
Продемонстрируйте клиенту детальный лог выполнения шагов:
* **Tool**: `harmony.autopilot.get_execution_log`
* Вывод покажет время работы каждого шага и статус верификации (например: `BG_Lab detected`, `Scientist rig imported`, `lipsync applied`).

## Шаг 4. Расчет бизнес-показателей
Запустите генератор отчета об экономии времени для демонстрации окупаемости инвестиций (ROI):
* **Tool**: `harmony.production.generate_time_savings_report`
* **Arguments**:
  ```json
  {
    "scenesCount": 10,
    "hourlyRate": 35
  }
  ```

## Что показать клиенту на презентации:
1. **Скорость**: Ручная сборка 10 сцен занимает 35 часов работы. Автопилот выполняет это пакетно за секунды.
2. **Аудит**: Автопилот автоматически проверяет результат на ошибки и битые связи.
3. **Безопасность**: Запуск в режиме `dry-run` или поддержка human-in-the-loop (запрос ручного подтверждения) гарантирует безопасность коммерческих файлов.
